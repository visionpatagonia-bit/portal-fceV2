  · Posiciones iniciales (lie down, stand up, sit, get into position)
  · Agarres (grip, grasp, hold) + tipos (overhand/underhand/neutral/wide/narrow)
  · Brazos/codos/manos (extend, bend, keep elbows close, palms facing)
  · Piernas/rodillas/pies (bend knees, feet shoulder-width)
  · Tronco/postura (keep back straight, engage core, chest up)
  · Acciones (lower/raise/push/pull/return slowly)
  · Repeticiones + pausas + respiracion (inhale/exhale)
  · Tips/precauciones (do not, avoid, be careful)
  · Anatomia (shoulders/chest/back/biceps/triceps/quads/hamstrings/glutes/calves)
  · Equipo (barbell/dumbbells/kettlebells/bench/cable/pulley/machine)
  · Conectores + direcciones (with/while/up/down/forward/backward)
- window.nxTranslateInstruction(s) · returns {text, wasTranslated}
- Heuristica: >=3 replacements + length > 30 → marca wasTranslated true
- Lightbox aplica traduccion per frase · sufijo (EN) discreto solo si NO traducida
- langTag "EN" en header del bloque solo si NADA traducido (UX clara)

DECISIONES
- Bundle yuhonas YA tiene nombre_es/equipamiento/musculos en ES → match directo ES↔ES
- Sinonimos blandos (banca/banco, dominada/dominadas) cubren plurales basicos
- Threshold 0.40 conservador · sacrifica recall por precision (evita false positives)
- 42% matching automatico aceptable para MVP · 58% restante son ejercicios curados
  muy especificos del Method Anubis (sin equivalente directo yuhonas)
- yuhonas_thumb URL inline en EX entries (no requiere bundle cargado para mostrar thumb)
- nxOpenExDetail async para lazy load NX_CATALOG si curado tiene match
- Diccionario manual cubre ~50% frases comunes · alta precisión · zero dependency

TRADE-OFFS
- 58% curados sin match (sin foto) · queda groupIcon · UX consistente
- Matching automatico puede tener false positives en low tier (0.40-0.59)
  → si Ariel reporta match incorrecto en demo, ajustar manualmente
- Traducciones literales en algunos casos · puede sonar levemente robotico
  → si frases especificas suenan mal, agregar patterns curados manualmente
